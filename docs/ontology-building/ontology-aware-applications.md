# Ontology-aware applications

Ontology-aware applications use operational objects and relationships as their native interface, instead of treating the platform as a set of disconnected endpoints.

## OpenFoundry mapping

The current repo already points in that direction through:

- `apps/web/src/routes/ontology`
- `apps/web/src/routes/workflows`
- `apps/web/src/routes/geospatial`
- `apps/web/src/routes/reports`
- `apps/web/src/routes/apps`
- `services/application-composition-service`

## What this enables

- richer object-centric UIs
- context-aware workflows
- safer action execution
- linked exploration across data, maps, and reports
- AI and analytics grounded in business objects
