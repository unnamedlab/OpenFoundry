# Simulation and what-if analysis

Simulation is one of the strongest current differentiators in the ontology backend shape.

## What exists today

- rule simulation endpoints
- action `what-if` branches
- object simulation endpoints
- scenario simulation endpoints

## Why this matters

This is exactly the kind of capability that lets an operational ontology become decision-support infrastructure rather than only a data catalog.

## OpenFoundry current vs target

| Topic | Current | Target |
| --- | --- | --- |
| rule simulation | route support exists | richer UI-backed previews |
| action what-if | branch management exists | full scenario authoring and comparison |
| object simulation | backend endpoints exist | application-integrated planning experiences |
