# Properties

Properties define the shape, semantics, and presentation behavior of ontology objects.

## What this subtree covers

- structural fields on object types
- formatting and display semantics
- required and edit-only fields
- derived or computed behaviors

## Repository signals

OpenFoundry already has dedicated property handlers and models in `ontology-service`, including route wiring for create, list, update, and delete operations on object-type properties.

## Why this matters

Properties are where raw source data becomes operationally meaningful and application-friendly.
