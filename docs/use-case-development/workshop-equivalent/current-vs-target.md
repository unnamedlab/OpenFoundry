# OpenFoundry current vs target

| Dimension | OpenFoundry current | OpenFoundry target |
| --- | --- | --- |
| app authoring | backend-managed app/page/widget lifecycle | richer visual composition experience |
| templates | already supported in app builder | broader catalog with governance and reuse |
| publish flow | versions and publish endpoints exist | promotion, branching, and environment-aware delivery |
| packaging | slate import/export and public load exist | tighter integration with marketplace and repos |
