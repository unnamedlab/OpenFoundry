# Derived and computed properties

Derived properties are where ontology semantics meet query logic, function logic, and automation.

## Why they matter

They let teams expose high-value operational information without forcing every application to recompute it manually.

## OpenFoundry current signals

The repo already contains building blocks that can support this direction:

- ontology functions
- query service
- rules and simulation
- quiver visual functions
- smoke scenarios with metrics and semantic transformations

## OpenFoundry current vs target

| Dimension | Current signals | Target capability |
| --- | --- | --- |
| computation | distributed across services | explicit derived-property runtime |
| authoring | implicit in logic and handlers | managed derived-property authoring |
| consumption | app and query driven | reusable semantic metric layer |
