# P3 governance flow

The P3 smoke scenario is the best current end-to-end proof that ontology semantics, objects, and governance logic can work together.

## Simplified sequence

```text
define interface and case type
  -> add semantic properties
  -> create supporting analyst and case objects
  -> attach rules and governed state transitions
  -> query and operate on the resulting semantic graph
```

## Why this matters

This gives us a concrete documentation anchor for how OpenFoundry’s ontology layer is meant to support real workflows rather than isolated CRUD.
