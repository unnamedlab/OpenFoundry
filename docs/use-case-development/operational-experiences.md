# Operational experiences

Operational experiences are the end-user surfaces where multiple platform capabilities meet.

## Current route signals

The frontend already exposes several route families that look like experience surfaces rather than generic admin pages:

- `apps/web/src/routes/ontology`
- `apps/web/src/routes/geospatial`
- `apps/web/src/routes/reports`
- `apps/web/src/routes/queries`
- `apps/web/src/routes/apps`
- `apps/web/src/routes/workflows`

## Why this matters

Use cases in OpenFoundry are likely to be delivered as composed experiences that combine:

- ontology-backed objects
- analytics and reports
- maps and graph exploration
- workflows and actions
- role-aware access patterns

This page is the bridge between capability documentation and solution design documentation.
