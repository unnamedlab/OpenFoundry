# Foundry feature parity matrix

Date: 2026-05-18
Machine-readable source: [foundry-feature-parity-matrix.json](./foundry-feature-parity-matrix.json)

This matrix closes checklist item `W0.1` in
[foundry-workshop-pipeline-1to1-checklist.md](./foundry-workshop-pipeline-1to1-checklist.md).
It maps public Palantir Foundry concepts to the current OpenFoundry surfaces,
with evidence links to source files, tests or test-gap tracking, and official
Palantir documentation.

Status values:

| Status | Meaning |
| --- | --- |
| `done` | Implemented, tested, documented, and verified through API or UI evidence. |
| `partial` | Some OpenFoundry surface exists, but behavior is incomplete or not wired end-to-end. |
| `todo` | No meaningful implementation found in the current repository inventory. |
| `blocked` | Implementation exists behind an unavailable dependency, missing runtime wiring, or an unmounted route. |

## Module-level parity summary

Per-module roll-up across the 34 `foundry-*-1to1-checklist.md` files. The
fine-grained per-feature table below still drives Pipeline Builder, Workshop,
Geospatial, Data Connection, Ontology Actions, and Functions; this summary
keeps the rest of the platform from showing up as silent gaps.

Counts were refreshed on 2026-05-18 for Functions Runtime and Global
Branching and originally collected on 2026-05-17 for the rest by inspecting
service handler/test files and
`services/edge-gateway-service/internal/proxy/router_table.go`. "Routes"
is `grep -rE '\.(Get|Post|Put|Delete|Patch)\(' services/<svc>/internal/ | wc -l`
across the listed services (test files excluded). "Done / total" counts only
checklist items where a handler file *and* a corresponding `_test.go` could be
identified; partial wiring is reported in the notes column rather than the
percentage. Items flagged `needs-verification` by the audit were treated as
not done for the headline percentage.
