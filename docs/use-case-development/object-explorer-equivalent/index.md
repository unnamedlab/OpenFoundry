# Object Explorer equivalent

OpenFoundry does not currently expose a product literally named Object Explorer, but several route families indicate an equivalent object-centric exploration layer is emerging.

## Current route signals

- `apps/web/src/routes/ontology`
- `apps/web/src/routes/queries`
- `apps/web/src/routes/datasets`
- `apps/web/src/routes/quiver`
- `apps/web/src/routes/lineage`

## What this capability should become

- search for operational objects
- save and revisit explored cohorts
- compare sets and pivots
- jump across graph, query, map, and workflow contexts

## Why this matters

For an ontology-driven platform, an explorer experience is often the default interface between users and the operational model.
