# Applications

Ontology-driven applications are where the semantic model becomes user-facing operational software.

## Candidate application families in OpenFoundry

- object exploration and search
- dashboards and analytics
- maps and geospatial workflows
- workflows and decision support
- internal apps built with the app-builder surface

## Repository mapping

- `apps/web/src/routes/datasets`
- `apps/web/src/routes/ontology`
- `apps/web/src/routes/geospatial`
- `apps/web/src/routes/reports`
- `apps/web/src/routes/workflows`
- `apps/web/src/routes/apps`

## Why this section exists

In capability-oriented docs, application pages are the bridge between semantics and actual user workflows. That is especially important for an operational platform like OpenFoundry.
